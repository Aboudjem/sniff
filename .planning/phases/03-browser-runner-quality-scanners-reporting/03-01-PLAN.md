**1. Create `src/browser/page-hooks.ts`** — PageHookPipeline that captures console errors (E2E-04), network failures (E2E-05), and screenshots on failure (E2E-03, D-15, D-16, D-17):

```typescript
import type { BrowserFinding } from '../core/types.js';
// Lazy-import playwright types only

export interface PageHook {
  name: string;
  setup(page: import('playwright').Page, viewport: string): void;
  collect(): BrowserFinding[];
  reset(): void;
}
```

Implement three hooks:

**ConsoleErrorHook**: Listens via `page.on('console', msg => ...)` for `msg.type() === 'error'` and `page.on('pageerror', err => ...)`. Each captured error becomes a `BrowserFinding` with `ruleId: 'e2e/console-error'`, severity `high`, the error message text, `url` set to the current page URL, `viewport` from parameter. `fixSuggestion: 'Check browser console for stack trace details.'`.

**NetworkFailureHook**: Listens via `page.on('response', res => ...)` for status codes >= 400. Also listens via `page.on('requestfailed', req => ...)` for failed requests. Each becomes a `BrowserFinding` with `ruleId: 'e2e/network-failure'`, severity `medium` for 4xx, `high` for 5xx, `critical` for failed requests. Message includes URL and status code. `fixSuggestion` should say `'Verify the resource URL is correct and the server is responding.'`.

**ScreenshotHook**: Does NOT listen to events. Instead, exposes an async method `captureFailure(page, name, outputDir)` that calls `page.screenshot({ path: join(outputDir, name + '.png'), fullPage: false })` and returns the path. This is called by the runner when a test assertion fails.

**PageHookPipeline class**: Has `register(hook)`, `setupAll(page, viewport)` that calls `setup` on each hook, `collectAll()` that returns all findings from all hooks, and `resetAll()`. Follow the ScannerRegistry error isolation pattern -- if one hook throws, others still run.

export interface DiscoveredElement {

**3. Create `src/browser/runner.ts`** — BrowserRunner class (D-01, D-02, D-04):

```typescript
import type { BrowserRunContext, BrowserRunResult, PageVisitResult } from './types.js';
import type { BrowserScanContext, BrowserScanner, ScanResult } from '../scanners/types.js';
import type { BrowserFinding } from '../core/types.js';
import type { SniffConfig } from '../config/schema.js';
```

Class `BrowserRunner`:
- Constructor takes `config: SniffConfig`
- Private field for browser scanners: `BrowserScanner[]`
- Method `registerScanner(scanner: BrowserScanner): void`
- Method `async run(ctx: BrowserRunContext): Promise<BrowserRunResult>`:
  1. Lazy-import: `const { chromium } = await import('playwright');`
  2. Launch browser: `const browser = await chromium.launch({ headless: ctx.headless, slowMo: ctx.slowMo });`
  3. Wrap in try/finally to ensure `browser.close()`.
  4. For each viewport in `ctx.viewports`:
     a. `const context = await browser.newContext({ viewport: { width: vp.width, height: vp.height } });`
     b. `const page = await context.newPage();`
     c. Create a `PageHookPipeline`, register ConsoleErrorHook and NetworkFailureHook, call `setupAll(page, vp.name)`.
     d. For each URL derived from `ctx.baseUrl` + routes (passed in `ctx.testFiles` or discovered):
        - `await page.goto(url, { waitUntil: 'networkidle', timeout: ctx.timeout });`
        - Build `BrowserScanContext` with page, viewport, baseUrl, config, rootDir.
        - Run each registered `BrowserScanner` in order (per D-04: E2E first, then a11y, then visual, then perf). Use try/catch per scanner for error isolation.
        - Collect hook findings via `pipeline.collectAll()`.
        - If any scanner found a failure (severity critical or high), capture screenshot via ScreenshotHook.
        - Build `PageVisitResult` with url, viewport name, findings, screenshotPath, duration.
        - Call `pipeline.resetAll()`.
     e. Close context.
  5. Return `BrowserRunResult` with all scan results, page visits, total duration, and unique URLs.

- Method `async close(): Promise<void>` for cleanup.

Use `performance.now()` for all timing. Use `SniffError` for domain errors (e.g., browser launch failure wraps with code `BROWSER_LAUNCH_FAILED`, unreachable URL wraps with code `URL_UNREACHABLE`).

All heavy imports (playwright) must use lazy `await import(...)` pattern. All relative imports use `.js` extension.
