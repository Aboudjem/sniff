The function returns a complete HTML document string. Build it from template functions (per UI-SPEC component inventory):

**Main export:**
```typescript
export function generateHtmlReport(report: SniffReport): string {
  return `<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Sniff QA Report</title>
  <style>${getStyles()}</style>
</head>
<body>
  ${renderHeader(report.metadata, report.summary)}
  ${renderNav(report.summary.byScanner)}
  <main>
    ${renderSections(report)}
  </main>
  ${renderFooter(report.metadata.version)}
</body>
</html>`;
}
```

**Template functions (all private, not exported):**

**`getStyles()`** — Returns CSS string with:
- CSS custom properties for light mode colors from UI-SPEC:
  ```css
  :root {
    --bg-primary: #FFFFFF;
    --bg-secondary: #F6F8FA;
    --accent: #2F81F7;
    --text-primary: #1C2128;
    --text-secondary: #57606A;
    --severity-critical: #DA3633;
    --severity-high: #DA3633;
    --severity-medium: #D29922;
    --severity-low: #3FB950;
    --severity-info: #8B949E;
    --font-body: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif;
    --font-mono: 'SF Mono', 'Fira Code', 'Cascadia Code', Consolas, monospace;
  }
  ```
- Dark mode override via `@media (prefers-color-scheme: dark)`:
  ```css
  @media (prefers-color-scheme: dark) {
    :root {
      --bg-primary: #0D1117;
      --bg-secondary: #161B22;
      --text-primary: #E6EDF3;
      --text-secondary: #8B949E;
    }
  }
  ```
- Body: `font-family: var(--font-body); font-size: 14px; line-height: 1.5; background: var(--bg-primary); color: var(--text-primary); max-width: 1200px; margin: 0 auto; padding: 64px 16px;`
- Spacing scale from UI-SPEC: xs=4px, sm=8px, md=16px, lg=24px, xl=32px, 2xl=48px, 3xl=64px
- Typography from UI-SPEC: body=14px/400, label=12px/600, heading=20px/600, display=28px/600
- Finding card: `border-left: 4px solid; background: var(--bg-secondary); padding: 16px; margin-bottom: 8px; border-radius: 4px;`
- Severity badge: `display: inline-block; padding: 2px 8px; border-radius: 12px; font-size: 12px; font-weight: 600;` with badge colors per severity from UI-SPEC
- Nav: `display: flex; gap: 8px; flex-wrap: wrap; padding: 16px 0; border-bottom: 1px solid var(--bg-secondary);`
- Nav links: `padding: 8px 16px; text-decoration: none; color: var(--accent); border-radius: 4px;`
- Visual diff: `.diff-grid { display: grid; grid-template-columns: repeat(3, 1fr); gap: 8px; }` with `@media (max-width: 768px) { .diff-grid { grid-template-columns: 1fr; } }`
- Performance table: `width: 100%; border-collapse: collapse;` with `.over-budget { background: #DA363310; }`
- Print: `@media print { nav { display: none; } details { open: true; } .card { break-inside: avoid; } }`
- `scroll-behavior: smooth;` on html element
- `details summary { cursor: pointer; }`
- Responsive: performance table gets `overflow-x: auto;` wrapper

**`renderHeader(metadata, summary)`** — Per UI-SPEC:
- Display title: "Sniff QA Report" at 28px semibold
- Subtitle: `Generated ${metadata.timestamp} | ${summary.pagesScanned} pages scanned | ${summary.total} issues found`
- Summary badges row: For each severity in order [critical, high, medium, low, info], if count > 0, render a badge: `<span class="badge badge-${severity}">${count} ${severity}</span>`
- If total === 0, render: `<h2>Clean Bill of Health</h2><p>No issues found across ${summary.pagesScanned} pages. Either your app is flawless or sniff missed something. Run again with --verbose to be sure.</p>`

**`renderNav(byScanner)`** — Jump links:
- Map scanner names to display names: `{ 'source': 'Source Code', 'accessibility': 'Accessibility', 'visual': 'Visual Regression', 'performance': 'Performance', 'e2e': 'Browser Tests' }`
- For each scanner with findings: `<a href="#section-${name}">${displayName}</a>`

**`renderSections(report)`** — For each scanner with findings:
- Group findings by scanner: `findings.filter(f => f.ruleId.startsWith(prefix))`
- Scanner prefixes: `e2e/` -> Browser Tests, `a11y/` -> Accessibility, `visual/` -> Visual Regression, `perf/` -> Performance, `source/` -> Source Code
- Each section: `<section id="section-${name}"><h2>${displayName}</h2>${renderSeverityGroups(sectionFindings, report)}</section>`
- Between sections: 48px gap (2xl spacing)

**`renderSeverityGroups(findings, report)`** — Per severity in SEVERITY_ORDER:
- Group header: `<h3>${severity.toUpperCase()}: ${count} issues</h3>` (heading style)
- Then render each finding as a card

**`renderFindingCard(finding, report)`** — Per UI-SPEC finding card anatomy:
- Left border color by severity (use CSS class `card-${severity}`)
- Header row: severity badge + scanner type tag + viewport tag (if BrowserFinding with viewport) + URL
- Body: finding.message at body size
- Fix row: if `fixSuggestion` exists, render `<p class="fix"><strong>Fix:</strong> ${fixSuggestion}</p>`
- Screenshot: if `screenshotPath` exists and matching screenshot has base64, render inside `<details><summary>Screenshot</summary><img src="data:image/png;base64,${base64}" alt="${caption}" style="max-width:100%"></details>`
- Code snippet: if `snippet` exists and non-empty, render in `<pre><code>${htmlEscape(snippet)}</code></pre>` at 12px monospace

**`renderVisualDiff(finding, report)`** — For visual regression findings (ruleId starts with `visual/regression`):
- Three-column grid: Baseline | Current | Diff
- Parse paths from finding.snippet (which contains `Baseline: path\nCurrent: path\nDiff: path`)
- Look up base64 data from report.screenshots array
- Below images: pixel count and percentage from finding.message
- Responsive: stacks vertically below 768px

**`renderPerfTable(report)`** — For performance section:
- If `report.performanceMetrics` exists, render a table per URL
- Columns: Metric | Value | Budget | Status
- Status: green checkmark (unicode &#10003;) for pass, red X (unicode &#10007;) for fail
- Over-budget rows get class `over-budget`
- Get budgets from findings or default values

**`renderFooter(version)`** — Per UI-SPEC:
- `<footer><p>Generated by sniff v${version}. No sugar-coating, no excuses.</p></footer>`

**Helper: `htmlEscape(s: string): string`** — Replace `&`, `<`, `>`, `"` with HTML entities. Used for all user-generated content in HTML output.

**Brutal tone (RPT-06):** Use direct language throughout. The section headers, finding messages, and summary text should follow UI-SPEC tone guidelines:
- "This page is broken" not "This page may have issues"
- "Fix this before shipping" not "Consider addressing this"
- The empty state says "sniff missed something" not "everything looks great"
