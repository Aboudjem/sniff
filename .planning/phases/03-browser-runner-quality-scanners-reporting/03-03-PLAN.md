Create `src/scanners/performance/index.ts` implementing `BrowserScanner`.

**Important design note (from RESEARCH.md):** Lighthouse needs its own Chrome instance and cannot share Playwright's browser. The PerformanceScanner will NOT use `ctx.page` directly. Instead, it collects URLs during the browser run and runs Lighthouse separately. The scanner stores URLs to measure and runs Lighthouse in `teardown()` or via a dedicated `measureAll()` method called by the runner after Playwright closes.

**Class: `PerformanceScanner`**

This scanner has a non-standard lifecycle because Lighthouse cannot share Playwright's browser:
- It implements `BrowserScanner` but its `scan()` only records the URL for later measurement.
- It exposes `async measureAll(urls: string[]): Promise<ScanResult>` that runs Lighthouse sequentially after the Playwright browser is closed.

```typescript
export class PerformanceScanner implements BrowserScanner {
  name = 'performance';
  private urls: Set<string> = new Set();
  private config: SniffConfig | null = null;
```

- `async scan(ctx: BrowserScanContext): Promise<ScanResult>`:
  1. Store config reference: `this.config = ctx.config;`
  2. Record the URL: `this.urls.add(ctx.page.url());`
  3. Return empty result (actual measurement happens later): `return { scanner: this.name, findings: [], duration: 0, metadata: { urlsCollected: this.urls.size } };`

- `async measureAll(): Promise<ScanResult>`:
  1. `const start = performance.now();`
  2. If `this.config?.performance?.enabled === false`, return empty result.
  3. Lazy-import: `const lighthouse = (await import('lighthouse')).default;` and `const chromeLauncher = await import('chrome-launcher');`
  4. Get budgets: `const budgets = this.config?.performance?.budgets ?? { lcp: 2500, fcp: 1800, tti: 3800 };`
  5. Launch Chrome: `const chrome = await chromeLauncher.launch({ chromeFlags: ['--headless', '--no-sandbox'] });`
  6. Wrap in try/finally to ensure `chrome.kill()`.
  7. For each URL in `this.urls`:
     ```typescript
     const result = await lighthouse(url, {
       port: chrome.port,
       output: 'json',
       onlyCategories: ['performance'],
       formFactor: 'desktop',
     });
     ```
  8. Extract metrics from `result.lhr.audits`:
     - LCP: `result.lhr.audits['largest-contentful-paint']?.numericValue`
     - FCP: `result.lhr.audits['first-contentful-paint']?.numericValue`
     - TTI: `result.lhr.audits['interactive']?.numericValue`
  9. For each metric, compare against budget. If value exceeds budget:
     - Calculate overshoot: `const overshoot = ((value - budget) / budget) * 100;`
     - Map severity per RESEARCH.md:
       - `overshoot > 100` -> `critical`
       - `overshoot > 50` -> `high`
       - `overshoot > 25` -> `medium`
       - else -> `low`
     - Create `BrowserFinding`:
       ```typescript
       {
         ruleId: `perf/${metricName}`,
         severity,
         message: `${metricLabel} is ${Math.round(value)}ms, exceeds budget of ${budget}ms by ${Math.round(value - budget)}ms`,
         filePath: url,
         line: 0, column: 0,
         snippet: `${metricLabel}: ${Math.round(value)}ms (budget: ${budget}ms)`,
         url,
         viewport: 'desktop',
         fixSuggestion: buildPerfSuggestion(metricName, value, budget),
       }
       ```
  10. Also store raw scores in metadata for the report: `metadata: { metrics: { [url]: { lcp, fcp, tti, score } } }`
  11. Return `{ scanner: this.name, findings, duration: performance.now() - start, metadata }`.

**Helper function `buildPerfSuggestion(metric, value, budget)`:**
- `lcp`: `'Optimize largest content element. Consider lazy-loading below-fold images, preloading critical resources, or reducing server response time.'`
- `fcp`: `'Reduce time to first paint. Inline critical CSS, defer non-critical JavaScript, and minimize render-blocking resources.'`
- `tti`: `'Reduce JavaScript execution time. Split large bundles, defer non-essential scripts, and reduce main-thread work.'`

Handle Lighthouse errors gracefully: if Lighthouse throws for a URL, push a single `info` finding with the error message and continue to the next URL (per D-04: failures in one scanner don't block others).

Export `function buildReport(results: ScanResult[], metadata: Omit<ReportMetadata, 'version'>, screenshots: Screenshot[]): SniffReport`:
1. Flatten all findings: `const findings = results.flatMap(r => r.findings);`
2. Build `bySeverity`: iterate findings, count per severity. Initialize all severities to 0.
3. Build `byScanner`: iterate results, count findings per scanner name.
4. Calculate passRate: `results.filter(r => r.findings.length === 0).length / results.length` (handle div-by-zero).
5. Extract performanceMetrics from the performance scanner's result metadata if present.
6. Read version from package.json or hardcode `'0.1.0'` (use a try/catch readFile of `join(process.cwd(), 'node_modules/sniff/package.json')` falling back to `'0.1.0'`).
7. Return complete `SniffReport` object.

Export `async function saveReport(rootDir: string, report: SniffReport, formats: string[]): Promise<string[]>`:
1. Determine output dir: `const dir = join(rootDir, '.sniff', 'reports');`
2. `await mkdir(dir, { recursive: true });`
3. Generate timestamp slug: `const slug = report.metadata.timestamp.replace(/[:.]/g, '-').slice(0, 19);`
4. For each format in `formats`:
   - `'json'`: lazy-import `./json.js`, write to `report-${slug}.json`
   - `'junit'`: lazy-import `./junit.js`, write to `report-${slug}.xml`
   - `'html'`: lazy-import `./html.js`, write to `report-${slug}.html`
5. Return array of written file paths.

**4. Create `src/report/junit.ts`** — JUnit XML formatter (RPT-04):

Export `function generateJunitReport(report: SniffReport): string`.

Build XML via template literals (no library per RESEARCH.md). Structure:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<testsuites name="sniff" tests="{total}" failures="{failures}" errors="{errors}" time="{duration}">
```

Group findings by scanner name into `<testsuite>` elements:
```xml
<testsuite name="{scannerName}" tests="{count}" failures="{failCount}">
```

Each finding becomes a `<testcase>`:
```xml
<testcase name="{ruleId} on {filePath}" classname="{scannerName}">
  <failure message="{xmlEscape(message)}" type="{severity}"/>
</testcase>
```

For scanners with zero findings, emit a passing testcase:
```xml
<testcase name="{scannerName} passed" classname="{scannerName}"/>
```

**Critical:** XML-escape all user content (message, filePath, snippet) to prevent malformed XML. Implement `xmlEscape(s: string): string` that replaces `&` -> `&amp;`, `<` -> `&lt;`, `>` -> `&gt;`, `"` -> `&quot;`, `'` -> `&apos;`.

Failures are findings with severity critical or high. Errors are scanners that had metadata.error. Tests count = total findings + passing scanners.
