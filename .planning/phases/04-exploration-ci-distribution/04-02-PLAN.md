# Plan 04-02: Chaos Monkey Exploration Mode

**Phase:** 04 - Exploration + CI + Distribution
**Requirements:** EXPLR-01, EXPLR-02, EXPLR-03
**Depends on:** None (uses existing Phase 2-3 infrastructure; no dependency on Plan 04-01)

## Goal

Implement an AI-driven chaos monkey exploration mode that autonomously navigates an app, clicks interactive elements, fills forms with edge-case data (XSS, SQL injection, Unicode, boundary values), and reports findings with full structured JSON decision traces. The explorer reuses the existing `BrowserRunner` browser lifecycle pattern and `PageHookPipeline` for passive monitoring.

Create `src/exploration/edge-cases.ts` (per D-02, all hardcoded, no external dependency):

```typescript
export const EDGE_CASE_PAYLOADS = {
  xss: [
    '<script>alert(1)</script>',
    '"><img src=x onerror=alert(1)>',
    "javascript:alert(1)",
    '<svg/onload=alert(1)>',
    '<img src=x onerror=prompt(1)>',
  ],
  sqli: [
    "' OR '1'='1",
    "1; DROP TABLE users--",
    "' UNION SELECT NULL--",
    '" OR 1=1--',
  ],
  unicode: [
    '\u0000',
    '\uFFFD',
    '\u{1F4A9}',
    '\u202E',
    '\u200B',
  ],
  boundary: [
    '',
    ' ',
    'a'.repeat(10000),
    '-1',
    '0',
    '999999999999999',
    '1.7976931348623157e+308',
  ],
  specialChars: [
    '../../../etc/passwd',
    '%00',
    '\r\n',
    '${7*7}',
    '{{7*7}}',
  ],
} as const;

export type PayloadCategory = keyof typeof EDGE_CASE_PAYLOADS;

/** Select a payload for a given input type. Uses category rotation for coverage. */
export function selectPayload(inputType: string, stepIndex: number): { value: string; category: PayloadCategory } {
  const categories = Object.keys(EDGE_CASE_PAYLOADS) as PayloadCategory[];
  // Rotate through categories based on step index for breadth
  const categoryIndex = stepIndex % categories.length;
  const category = categories[categoryIndex];
  const payloads = EDGE_CASE_PAYLOADS[category];
  const payloadIndex = Math.floor(stepIndex / categories.length) % payloads.length;
  return { value: payloads[payloadIndex], category };
}
```

**Implementation:**

Create `src/exploration/page-state-extractor.ts`:
- Export `async function extractPageState(page: Page): Promise<PageState>` -- uses `page.evaluate()` to query the live DOM for:
  - All `a`, `button`, `[role="button"]`, `input[type="submit"]` elements -> `InteractiveElement[]`
  - All `input`, `select`, `textarea` elements -> `FormField[]`
  - For each element, extract: tag, text content (truncated to 100 chars), selector (prefer `[data-testid]` > `#id` > `[name]` > CSS path), type attribute, role attribute, name, required, placeholder.
  - Mark elements as `visited: false` (caller tracks visited state).
  - Use `page.title()` and `page.url()` for page-level state.

Create `src/exploration/runner.ts` -- `ExplorationRunner` class:
- Constructor: `(config: SniffConfig, explorationProvider: ExplorationProvider)`
- Main method: `async explore(ctx: ExplorationContext): Promise<ExplorationResult>`
- Browser lifecycle mirrors `BrowserRunner`: lazy `import('playwright')`, `chromium.launch({ headless: ctx.headless })`, try/finally with `browser.close()`.
- Create `PageHookPipeline` with `ConsoleErrorHook`, `NetworkFailureHook`, `ScreenshotHook` -- same as `BrowserRunner`.
- **Route seeding (D-01 — intelligent crawling via route-discoverer):** Before the main loop, call `discoverRoutes(ctx.rootDir, frameworks)` (import from `../../analyzers/route-discoverer.js`) to get all app routes. Detect frameworks first via `detectFrameworks(ctx.rootDir)`. Convert discovered route paths to absolute URLs by prepending `ctx.baseUrl` (e.g., `/dashboard` → `http://localhost:3000/dashboard`). Deduplicate with `ctx.baseUrl` itself as the first entry. Build a `pendingUrls: string[]` queue from this list. The main loop navigates each `pendingUrl` in order, runs `maxSteps` total steps across all URLs (not per-URL).
- Navigation: start with the first URL from `pendingUrls`. After exhausting steps on one URL, advance to the next pending URL. Use `page.goto(url, { waitUntil: 'networkidle', timeout: ctx.timeout })`.
- Main loop runs for `ctx.maxSteps` iterations:
  1. Call `extractPageState(page)` to get current interactive elements and form fields.
  2. Call `explorationProvider.decideNextAction(pageState, actionLog)` to get AI decision.
  3. Execute the decision:
     - `click`: `page.click(decision.selector, { timeout: 5000 })` wrapped in try/catch.
     - `fill`: `page.fill(decision.selector, decision.value!, { timeout: 5000 })` -- value comes from AI or from `selectPayload()` as fallback.
     - `navigate`: `page.goto(decision.url!, { waitUntil: 'networkidle', timeout: ctx.timeout })`.
     - `scroll`: `page.evaluate(() => window.scrollBy(0, 500))`.
     - `done`: break loop.
  4. After each action, collect hook findings via `pipeline.collectAll()`, count console errors and network failures for the observation log.
  5. Build `ExplorationActionLog` entry with step number, timestamp, url before action, action details, reasoning, and observation (url after, error counts, screenshot if severe finding).
  6. Call `pipeline.resetAll()` after collecting.
- After loop ends, return `ExplorationResult` with full action log, all findings, visited URLs, step count, and duration.
- Breadth-first priority (per RESEARCH recommendation): when building `PageState`, sort interactive elements with forms first > submit buttons > links not yet visited > other elements.

Create `src/exploration/claude-exploration-provider.ts`:
- Class `ClaudeExplorationProvider implements ExplorationProvider`.
- `decideNextAction(pageState, history)` builds a prompt describing current page state (URL, title, available elements, form fields) and recent history (last 5 actions), then calls Claude Code CLI using same pattern as `ClaudeCodeProvider`:
  ```typescript
  const { stdout } = await execFile('claude', [
    '--print', '--output-format', 'json',
    '--system-prompt', systemPrompt,
    userPrompt,
  ], { timeout: 30_000, maxBuffer: 1024 * 1024 });
  ```
- System prompt instructs Claude to respond with JSON matching `ExplorationDecision` shape. Emphasize: pick the most interesting untested element, prefer forms over navigation, use edge-case values for inputs, explain reasoning.
- Parse response, validate it has required fields, return `ExplorationDecision`. On parse error, return `{ action: 'done', reasoning: 'Failed to parse AI response' }`.
- Fallback: if Claude Code CLI is not available (`ENOENT`), use a deterministic heuristic: fill first unfilled form field with `selectPayload()`, click first unvisited button, navigate to first unvisited link, then `done`.

```typescript
export async function exploreCommand(options: {
  baseUrl?: string;
  maxSteps?: string;
  headless?: boolean;
  json?: boolean;
}): Promise<void> {
  const pc = (await import('picocolors')).default;
  const { loadConfig } = await import('../../config/loader.js');
  const { ExplorationRunner } = await import('../../exploration/runner.js');
  const { ClaudeExplorationProvider } = await import('../../exploration/claude-exploration-provider.js');
  const { saveResults } = await import('../../core/persistence.js');

  const config = await loadConfig(process.cwd());
  const baseUrl = options.baseUrl ?? config.browser?.baseUrl;
  if (!baseUrl) {
    console.error(pc.red('No base URL. Set browser.baseUrl in config or pass --base-url.'));
    process.exit(1);
  }

  // Validate URL (same pattern as run command T-03-13)
  try { new URL(baseUrl); } catch {
    console.error(pc.red(`Invalid base URL: "${baseUrl}".`));
    process.exit(1);
  }

  const maxSteps = options.maxSteps ? parseInt(options.maxSteps, 10) : (config.exploration?.maxSteps ?? 50);
  const provider = new ClaudeExplorationProvider();
  const runner = new ExplorationRunner(config, provider);

  console.log(`\nSniff v0.1.0 -- Chaos Monkey Exploration\n`);
  console.log(`  Target: ${baseUrl}`);
  console.log(`  Max steps: ${maxSteps}\n`);

  const result = await runner.explore({
    baseUrl,
    rootDir: process.cwd(),
    headless: options.headless ?? config.browser?.headless ?? true,
    maxSteps,
    timeout: config.exploration?.timeout ?? config.browser?.timeout ?? 30000,
    viewport: config.exploration?.viewport ?? { width: 1280, height: 720 },
  });

  // Save exploration log to .sniff/exploration-<timestamp>.json
  const { writeFile, mkdir } = await import('node:fs/promises');
  const { join } = await import('node:path');
  const dir = join(process.cwd(), '.sniff');
  await mkdir(dir, { recursive: true });
  const logPath = join(dir, `exploration-${new Date().toISOString().replace(/[:.]/g, '-')}.json`);
  await writeFile(logPath, JSON.stringify({ actionLog: result.actionLog, findings: result.findings }, null, 2));

  // Convert findings to ScanResult format for persistence
  const scanResult = {
    scanner: 'exploration',
    findings: result.findings,
    duration: result.duration,
    metadata: { pagesVisited: result.pagesVisited, totalSteps: result.totalSteps },
  };
  await saveResults(process.cwd(), [scanResult]);

  // Terminal output
  if (options.json) {
    console.log(JSON.stringify(result, null, 2));
  } else {
    console.log(`\n  Steps completed: ${result.totalSteps}`);
    console.log(`  Pages visited: ${result.pagesVisited.length}`);
    console.log(`  Findings: ${result.findings.length}`);
    console.log(`  Action log: ${logPath}\n`);

    if (result.findings.length > 0) {
      const { formatBrowserFindings } = await import('../formatter.js');
      console.log(formatBrowserFindings(result.findings));
    }
  }
}
```

## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI response -> action execution | AI decides what to click/fill; malformed response could cause unexpected navigation |
| Edge-case payloads -> target app | XSS/SQLi payloads are intentionally hostile data sent to the target application |
| Target app -> Playwright page | Responses from target app are rendered in browser context |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-04-05 | Spoofing | AI response parsing | mitigate | Validate `ExplorationDecision` shape before executing. Reject unknown action types. Default to `done` on parse failure. |
| T-04-06 | Tampering | Navigation URLs from AI | mitigate | Validate AI-suggested URLs against `baseUrl` origin before `page.goto()`. Reject cross-origin navigation. Same pattern as BrowserRunner T-03-01. |
| T-04-07 | Denial of Service | Exploration loop | mitigate | Hard cap on `maxSteps` (default 50). Per-action timeout of 5000ms. Per-navigation timeout via `ctx.timeout`. Browser closed in finally block. |
| T-04-08 | Information Disclosure | Edge-case payloads in logs | accept | Payloads are intentionally visible in exploration logs for debugging. Logs are local `.sniff/` files. No secrets involved. |
| T-04-09 | Elevation of Privilege | Arbitrary selector execution | mitigate | Selectors come from AI or live DOM extraction. Playwright `page.click()`/`page.fill()` operate within browser sandbox. No shell execution from selectors. |
</threat_model>
