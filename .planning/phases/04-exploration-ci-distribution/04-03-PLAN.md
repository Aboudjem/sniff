# Plan 04-03: CI Integration (`sniff ci` + GitHub Actions)

**Phase:** 04 - Exploration + CI + Distribution
**Requirements:** CI-01, CI-02
**Depends on:** Plan 04-01 (flakiness detection must exist for CI exit-code quarantine logic; `--ci` flag on `run` command already wired in 04-01)

## Goal

Implement the `sniff ci` command that generates an opinionated GitHub Actions workflow file (`.github/workflows/sniff.yml`) with Playwright browser caching, headless run, report artifact upload, and proper exit codes. CI mode detection is already wired in Plan 04-01 (the `--ci` flag and `CI` env var handling on the `run` command); this plan adds the workflow generation command itself.

## Context

**Existing code this plan builds on:**

- `src/cli/index.ts` -- Commander program with existing commands (`init`, `scan`, `run`, `report`, `update-baselines`). Plan 04-01 adds `--ci` and `--track-flakes` flags to `run`. Plan 04-02 adds `explore`. This plan adds the `ci` command.
- `src/cli/commands/run.ts` -- After Plan 04-01, the run command handles CI mode: forces headless, JUnit XML format, flakiness tracking, quarantine-aware exit codes (D-10). This plan does NOT modify the run command.
- `package.json` -- Current name is `sniff` but npm name is taken (RESEARCH Risk 1). The workflow template must use a configurable package name. Default to `sniff-qa` per RESEARCH recommendation, but accept override.

**Key interfaces (from Plan 04-01):**

```typescript

program.command('run')

**GitHub Actions workflow template specifics (per RESEARCH, verified):**
- `ubuntu-latest` runner, Node 22, `actions/cache@v4` for Playwright browsers at `~/.cache/ms-playwright`

- `if: always()` on artifact upload so reports persist even on failure
- `timeout-minutes: 15` to guard against hung Playwright sessions

export interface CIWorkflowOptions {
  packageName: string;       // npm package name for npx command (default: 'sniff-qa')
  nodeVersion: string;       // default: '22'
  branches: string[];        // default: ['main', 'master']
  timeoutMinutes: number;    // default: 15
  retentionDays: number;     // default: 30
  packageManager: 'npm' | 'pnpm' | 'yarn';  // auto-detected or specified
}
```

Create `src/ci/workflow-template.ts`:
- Export `function generateGitHubActionsWorkflow(options: CIWorkflowOptions): string` -- returns the complete YAML string per RESEARCH section 3 template. Key details:
  - `name: Sniff QA`
  - Triggers: `push` and `pull_request` on configured branches
  - Single `sniff` job on `ubuntu-latest` with `timeout-minutes`
  - Steps: checkout@v4, setup-node@v4 with cache, install deps (npm ci / pnpm install / yarn install based on `packageManager`), cache Playwright browsers at `~/.cache/ms-playwright` keyed on `runner.os` + lockfile hash, conditional Playwright install (with-deps on miss, install-deps on hit), run sniff with `npx ${options.packageName} run --ci` and `CI: true` env, upload artifact with `if: always()` for `sniff-reports/` directory.
  - For pnpm: add `pnpm/action-setup@v4` step before setup-node. For yarn: use `yarn install --frozen-lockfile`.
- Export `function detectPackageManager(rootDir: string): Promise<'npm' | 'pnpm' | 'yarn'>` -- check for `pnpm-lock.yaml`, `yarn.lock`, or `package-lock.json` in `rootDir`. Default to `npm`.

Create `src/cli/commands/ci.ts`:

```typescript
export async function ciCommand(options: {
  output?: string;
  force?: boolean;
  packageName?: string;
}): Promise<void> {
  const pc = (await import('picocolors')).default;
  const { existsSync } = await import('node:fs');
  const { mkdir, writeFile } = await import('node:fs/promises');
  const { join, dirname } = await import('node:path');
  const { generateGitHubActionsWorkflow, detectPackageManager } = await import('../../ci/workflow-template.js');

  const outputPath = options.output ?? '.github/workflows/sniff.yml';

  // Validate output path (T-04-10 mitigation)
  if (!outputPath.endsWith('.yml') && !outputPath.endsWith('.yaml')) {
    console.error(pc.red('Output path must end with .yml or .yaml'));
    process.exit(1);
  }
  const fullPath = join(process.cwd(), outputPath);
  if (!fullPath.startsWith(process.cwd() + '/') && fullPath !== process.cwd()) {
    console.error(pc.red('Output path must be within the project directory'));
    process.exit(1);
  }

  // Check for existing file
  if (existsSync(fullPath) && !options.force) {
    console.error(pc.yellow(`Workflow already exists at ${outputPath}. Use --force to overwrite.`));
    process.exit(1);
  }

  const packageManager = await detectPackageManager(process.cwd());
  const packageName = options.packageName ?? 'sniff-qa';

  const yaml = generateGitHubActionsWorkflow({
    packageName,
    nodeVersion: '22',
    branches: ['main', 'master'],
    timeoutMinutes: 15,
    retentionDays: 30,
    packageManager,
  });

  await mkdir(dirname(fullPath), { recursive: true });
  await writeFile(fullPath, yaml, 'utf-8');

  console.log(`\n${pc.green('Created')} ${outputPath}\n`);
  console.log(`  Package manager: ${packageManager}`);
  console.log(`  Package name: ${packageName}`);
  console.log(`  Triggers: push + PR to main/master`);
  console.log(`  Caches: Playwright browsers + node_modules`);
  console.log(`\n${pc.dim('The workflow runs')} ${pc.bold('npx ' + packageName + ' run --ci')} ${pc.dim('which enables:')}`);
  console.log(`  ${pc.dim('- Headless browser mode')}`);
  console.log(`  ${pc.dim('- JUnit XML output')}`);
  console.log(`  ${pc.dim('- Flakiness tracking + quarantine')}`);
  console.log(`  ${pc.dim('- Non-zero exit on failure')}\n`);
}
```

Modify `src/cli/index.ts` -- add before `await program.parseAsync()`:

## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI options -> file system write | `--output` path controls where the YAML file is written |
| Generated YAML -> CI runner | Workflow file is executed by GitHub Actions with repository permissions |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-04-10 | Tampering | --output path | mitigate | The output path is relative to `process.cwd()`. Validate it ends with `.yml` or `.yaml`. Use `path.join()` to prevent path traversal. Do not allow absolute paths outside the project. |
| T-04-11 | Tampering | Generated workflow content | accept | Workflow YAML is a static template with only the package name, node version, and branches as variables. No user-controlled code injection vector -- all variables are simple strings inserted into known YAML fields. |
| T-04-12 | Information Disclosure | Workflow file in repo | accept | The workflow file is intentionally committed to the repo. It contains no secrets (secrets are configured in GitHub repo settings, not in the YAML). |
| T-04-13 | Denial of Service | CI runner timeout | mitigate | Workflow includes `timeout-minutes: 15` to prevent runaway Playwright processes from consuming CI minutes indefinitely. |
