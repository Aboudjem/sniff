}

1. Create `src/ai/response-parser.ts` — export function `parseGeneratedTest(rawOutput: string, routePath: string): GeneratedTest`:
   - Extract code block: regex match ` ```(?:typescript|ts)\n([\s\S]*?)``` ` from rawOutput
   - Fallback: if no fenced block found but rawOutput contains `import { test`, treat entire rawOutput as code
   - If no code found at all: throw `new SniffError('PARSE_NO_CODE', 'AI response contained no TypeScript code block')`
   - Validate: check extracted code contains `import { test` or `import { expect` pattern. If not: throw `new SniffError('PARSE_INVALID_TEST', 'Generated code does not contain Playwright test imports')`
   - Extract reasoning: scan code for JSDoc comments (`/** ... */`) above `test(` calls. Concatenate all JSDoc content as the `reasoning` string.
   - Return `{ specContent: extractedCode, reasoning, route: routePath }`

2. Create `src/ai/provider.ts` — export function `resolveProvider(): AIProvider`:

   - If not set: lazy-import `./claude-code.js` and return `new ClaudeCodeProvider()`
   - Use dynamic `import()` for both providers (lazy-load pattern per D-03 from Phase 1)
   - For now, since the actual provider classes don't exist yet (Plan 04), export the function signature and use lazy imports so it compiles. The function returns `Promise<AIProvider>` (async due to lazy imports).

| env vars -> provider resolver | Environment variable controls which provider is instantiated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
